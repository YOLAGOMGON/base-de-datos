08 - Guia para la prueba de desempeno (modulo 4)

Objetivo
Tener una referencia clara de como suelen ser las pruebas practicas de
bases de datos y como prepararte con los temas del modulo.

Nota importante
No hay informacion publica oficial sobre el formato exacto de la prueba
de desempeno de Riwi. Esta guia se basa en practicas comunes de pruebas
tecnicas en academias de software y en los temas indicados en tu mensaje.

Formato tipico (referencia)
- Tiempo limitado (1 a 3 horas).
- Entrega de un repositorio con README y evidencias.
- Base de datos relacional (Postgres).
- API con CRUD y pruebas en cliente (Thunder o Postman).

Temas que suelen evaluar
1) Conexiones a Postgres y MongoDB
   - Conectar y ejecutar una consulta simple.
   - Entender credenciales y strings de conexion.
2) Excel normalizado
   - Convertir una hoja desordenada a tablas 1FN, 2FN, 3FN.
   - Definir PK y FK.
3) Cargar Excel a Postgres
   - Exportar a CSV y usar COPY o \copy.
4) Express CRUD + DB avanzada
   - CRUD completo y manejo de errores.
   - Procedimientos almacenados, vistas y triggers.
5) Probar la API (Thunder Client)
   - Evidencias de POST, GET, PUT, DELETE.
6) Documentacion en Postman
   - Coleccion exportada a JSON.
7) README + ERD
   - Que hace el proyecto, como corre y diagrama ERD.

Entregables tipicos
- Repositorio con codigo y README.
- Scripts SQL organizados.
- Coleccion Postman (JSON v2.1).
- ERD en PNG o PDF.
- Evidencias de pruebas (capturas o JSON).

Checklist de preparacion
- Puedo crear y relacionar tablas en Postgres.
- Puedo explicar normalizacion con un ejemplo.
- Puedo cargar CSV con COPY.
- Puedo crear procedimientos, vistas y triggers.
- Puedo levantar una API y probarla con Thunder.
- Puedo documentar en Postman y exportar JSON.
- Tengo README claro y ERD listo.

Recomendacion final
Si tienes acceso a plataforma interna (Moodle o guia oficial),
compárala con esta guia y ajusta el estudio a lo que te pidan.
